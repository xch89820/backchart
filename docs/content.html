Introduction

What is this? How does it help you?

What is this?

Firstly, the library is not a graphics library, it is just a MVC(Model View Controller) implementation for graphics library based on Backbone.If you not familiar the Backbone library, you should learn it at the beginning. The library only support CanvasJS at present, but will support more widespread use of chart libraries like Highcharts, D3 etc.The follow is a list that the library will support to in future.

name Rendering technique Has supported License of the library Website
CanvasJS HTML5 Canvas Yes CC http://canvasjs.com/
Highcharts VML ,SVG, HTML5 Canvas No CC http://www.highcharts.com/
D3 HTML, SVG No BSD http://d3js.org/
amcharts VML ,SVG No has limitation of the free version http://www.amcharts.com/
flotr2 HTML5 Canvas No MIT http://www.amcharts.com/

How does it help you?

The aim of the project is that give the traditional graphics library MVC properties based on Backbone contained data-driver, local storage etc.You can find more detail advances in here

Backchart will give you chart library :

  • Data-driver
  • Date shared
  • More ways to manipulate data
  • Local storage
  • Concise code logic
  • more...

Browser Compatibility

  • IE 6+*
  • Chrome 8+
  • Firefox 10+
  • Safari 3+
  • Opera 10.6+
*Support for IE depends on you chart library.You can use excanvas to make the IE6-8 to support HTML Canvas

Getting Started

How to install, import and use it.

Install

If you async loading your modules by AMD or CommonJS ,please copy all JavaScript files in dist directory but not browser directory to your library location. Otherwise, please copy all files in browser directory to your target.

Import

At the beginning, you should import Backbone, Underscore and the chart libraries has be supported in your application.

AMD

Add backchart path in your path config.The follow code is a example loading Backchart CanvasJS library.

require.config({
   path :{
       backbone : "you/backbone/path",
       underscore : "you/underscore/path",

       //import the chart library you want to use.
       CanvasJS : "../libs/charts/canvasjs/canvasjs",

       //import the backchart to support your chart library
       "backchart" : "backchart.canvasjs"
       //...code...
   }
});

You can import different types of libraries in your application , and make true every type of chart has a unique name in your path config.

require.config({
   path :{
       //import the backchart to support your chart library
	   "main.backchart" : "backchart.highcharts",
	   "backup.backchart" : "backchart.canvasjs"
       //...code...
   }
});

Get the Model,Collection and View instance by require backchart.CHARTNAME

//Define you model
define(["CanvasJS","backchart"], function(){
	require(["backchart.canvasjs"], function(chart){
		var backchartModel = backchart.model,
            backchartCollection = backchart.collection,
			backchartView = backchart.view;
		//do something..
	});
});

Simple usage

Firstly, you should has a few understanding of MVC structure and Backcbone library. Backchart use Backbone to wrap the processing of synchronous data and store.You don't need to care about rendering when your data changed because any changed will drive the view layer(you chart element) redraw immediately.

We still use CanvasJS to begin our example.The collection configure is same as the data configure in CanvasJS and the data format same as too.

//Define the view inherit to backchart.canvasjs.view;
var myView = new backchartView.extend({
	container : document.getElementById("simple-example")
});
//Define the collection
var myCollection = new backchartCollection;

//Bind collection to your view and set the render configure
myView.onCollection(myCollection, {
	type: "column",
	showInLegend: true,
	legendText: "The column"
});
//You can bind the collection again by another configure
myView.onCollection(myCollection, {
    type: "line",
    showInLegend: true,
	legendText: "The line"
});
//Set the data
myCollection.set([
	{ x: 10, y: 97800, label: "UAE"},
	{ x: 20, y: 267017,  label: "Saudi Arabia" },
	{ x: 30, y: 116000,  label: "Russia"},
	{ x: 40, y: 297571, label: "Venezuela"},
	{ x: 50, y: 154580,  label: "Iran"},
	{ x: 60, y: 175200,  label: "Canada"},
	{ x: 70, y: 20682,  label: "US"},
	{ x: 80, y: 20350,  label: "China"}
]);

As you see, we get two types of chart by the same collection, and you just need to set data to the collection and you will get the chart.Backchart will do all thing automatically.

BEA Example

We use the data from the Bureau of Economic Analysis (BEA) publishes economic statistics API as the data source, and we will explain a more complex example which dealing with the data to you and describe the code which can help you to have a basic understanding how to use the Backchart library.

BEA API support JSONP and CORS features, we use JSONP in here.

Assuming that one day you boss want you to make a report which showing the Per Capita Personal Income of the U.S. states in company's MIS.You begin to work out it with Backchart.

//Define the view inherit to backchart.canvasjs.view;
var BEAView = new (backchartView.extend({
    container : document.getElementById("bea-example")
}));
/**
 * Define the model to deal with each row of data
 * You should covered each data received to your chart can accept.
 * We used CanvasJS, so add label,legendText and y in my data
 **/
var BEAModel = backchartModel.extend({
	parse : function(respo){
    	return {
            label : respo.GeoName,
            legendText: respo.GeoName,
            y : parseInt(respo.DataValue,10)
        }
	}
});
//Define collection and set the URL and the name of dataset in the result
var BEACollection = new (backchartCollection.extend({
     model: BEAModel,
     url: "http://www.bea.gov/api/data",
	 parse : function(respo){
		return respo.BEAAPI ? respo.BEAAPI.Results.Data : [];
     }
}));
//Bind collection to view
BEAView.onCollection(BEACollection, {
     type: "bar",
     showInLegend: true
});

//Fetch data
BEACollection.fetch({
     dataType: "jsonp",
     jsonp:"jsonp",
     data: {
           UserID : "2A28DB6F-5FE7-4A05-8899-D6F4B72C5102",
           method : "GetData",
           datasetname : "RegionalData",
           KeyCode : "PCPI_CI",
           GeoFIPS : "STATE",
           Year : "2011",
           ResultFormat : "json"
     }
});

In the above illustration ,we create the basic factors : Model, Collection and View.

In Model, we defined the method that how to covered format of each data in results. Model is the minimum data unit in Backchart and each model in collection represent a point or a column or other unit in your chart's data.In the CanvasJS, each object in dataPoints Array is as a model in Backchart.

The Collection is a dataset of models, you can set the URL to reference its location on the server.In the CanvasJS, each object in data Array is as a collection.

The View is a object wrapped the element up and will listen to the events from the collections whose have been bound to used onCollection function.You must pass a HTMLElement to the container in view and make sure it not be hidden.

As you see, when you invoke the fetch function to synchronous data from server, the chart will be draw on the element you passed to

Change chart type

After you finished all work and prepare to go home, the big boss tell you he want to check the percentage of each data by a pie chart.He just care about the data above the average and the reset of data should be add to an item called Other.Normally, we should change lots of code in you application because you should reconfigure the chart, but you just change a little in Backchart

var BEAView = new (backchartView.extend({
    container : document.getElementById("bea-example")
}))({
	//Add the toolTip configure in view, please see CanvasJS document("http://canvasjs.com/docs/charts/chart-options/tooltip/) to get more detail
	toolTip:{
       content: "{label} : {y} ({percent})"
   	}
});
BEACollection.on("sync", function(){
	var usAverageModel = BEACollection.findWhere({label: "United States"});
	var usAverage = usAverageModel.get("y");
	BEACollection.remove(usAverageModel);

	//First ,get the total value of data which bellow the average and all, filter this data at the same time.
	var otherValue = 0,
		totalValue = 0;
	var modelSet = BEACollection.filter(function(model){
		var val = model.get("y");
		totalValue += val;
		if (val < usAverage){
			otherValue += val;
			return false;
		}
		return true;
	});
	modelSet.push(new BEAModel({GeoName:"other",DataValue:otherValue})); 
	$.each(modelSet, function(index, model){
		model.set("percent", (model.get("y")/totalValue * 100).toFixed(2) + "%");
	});
		
	//Change the type
	BEAView.getRenderOptions(BEACollection).type = "pie";
	//Add the percentage to all models attribute
	BEACollection.reset(modelSet);
});

More example for describe the basic function and enumerate some useful APIs

Write your model and collection

At the beginning of creating your Backchart, you should define your Model and Collection extended from into Backchart.base that can describe the conversion mode from original data to data can be used by your chart library.

In MVC structure, Model containing the interactive data as well as a large part of the logic surrounding it and the collection is a logical grouping of date wrapped by Model. Make an CanvasJS example. You original data is here

{
   success:true,
   code:200,
   result:[
	   {index:"XA-01", group:"backchart,json-cover,upload",name:"Jone Casper", age: "24", phone:"13365772345"},
	   {index:"XA-02", group:"bk, jsMusic", name:"Steve Robin", age: "26", phone:"(912) 555-1234"},
       //...
   ]
}

You want to make a column chart about each person's age.Obviously only two fields name and age in the result is useful.So the first, we should define a model that only store the two filed and use it to initialize the lable and y filed (CanvasJs use "label" to indicate what a certain position on the axis means and y is the data value)

var myModel = backchartModel.extend({
	parse : function(respo){
		return {
			label : respo.name,
			y : parseInt(respo.age, 10)
		}
	}
});

And then, we should define how to convey the result filed from return data and use it to initialize our Collection. 

var myCollection = backchartCollection.extend({
	 model: myModel,
	 parse : function(respo){
		return respo.success ? respo.result : [];
	 }
}));

Create a view and bind collections

After defined the model and collection, a view instance inherit Backchart.CHART.view can help you to draw the chart by data in collection.

In Backbone, the default view is so simple that just implement the basic event model ,it is not enough to make a relationship between view and lots of collections, so we defined onCollection to bind collection to view and return a unique ID to identify the relation.

The onCollection function has three arguments need to. The first argument is "collection" ,please passing a collection instance, and the second is "renderOptions" which defined all dataset level properties like type ,name, etc. you should check the document of your chart before pass to.The third argument is "options" that defined the collection bind attributes like bid(Bind ID), visible, etc. The more detail API is here

After bind one collection to a view instance, the instance will change(redraw) when you change the content of the collection.Obviously, there are serious problem should be noticed:

What events will trigger the instance redraw?

By the default, those events will be listened to used to trigger if your collection inherited the Backchart.base.collection or it's upper class.

  • seted
  • removed
  • change
  • destroy
  • reseted
  • sort
  • change:visible

And the ordinary backbone collection will be listened

  • set
  • add
  • change
  • destroy
  • reset
  • sort

When have a lots of data insert to ,remove from the collection or you want to reset the collection,how to prevent to call the redraw function frequently?

If you collection inherited Backchart.base.collection or it's upper class, the default set, remove and reset function has be overwritten and will send seted, removed and reseted event after data operation finished and the redraw function will be called after this.For example

backCollection.add([
	{name:"Jone", age:35}, //send add event
	{name:"Harry", age:25}, //send add event
	{name:"Mark", age:33}  //send add event
]);
//send seted event and the chart will redraw at the same time

The collection in the above code will send 3 times of add events when the runtime and just send one time of seted event after finished.There are only one redraw behavior as the same time of seted occur.

var wanttoadded = [ 
	{name:"Jone", age:35}, 
	{name:"Harry", age:25},
	{name:"Mark", age:33}
]
//Avoid to do this!
for (var i=0; i<wanttoadded.length; i++){
	backCollection.add(wanttoadded[i]); //send add event and seted event
}

You should avoid to invoke add, set ,remove function in loop because it will make a lot of events and trigger the graph in view redraw many times.It will have a strong impact on the speed of your web application.

If you really want to use loop because you have some other things in this loop, call the setSilence(true) in your collection or your view to ban responding any events temporarily after the loop is a good idea and be recommended, and when jumping out of the loop, you should call the setSilence(false) to recover the default.

var wanttoadded = [ 
	{name:"Jone", age:35}, 
	{name:"Harry", age:25},
	{name:"Mark", age:33}
]
backCollection.setSilence(true);
for (var i=0; i<wanttoadded.length; i++){
	backCollection.add(wanttoadded[i]); //send add event and seted event
}
backCollection.setSilence(false);
//To trigger redraw
backCollection.trigger("seted");

Under some special condition, you want to rewrite the render function and call it when a given event be made.You can cover the default variable eventCallback when make the view instance.
